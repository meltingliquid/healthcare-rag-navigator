Recommended approach: Two Databricks Jobs
Job A runs your full pipeline once (data engineering ‚Üí indexes ‚Üí retrieval setup). Job B runs 07_query_pipeline.py independently any time. This is far better than manually running notebooks one by one.

Then update your notebooks to also fetch the OpenAI key the same way ‚Äî replace `os.getenv("OPENAI_API_KEY")` with `dbutils.secrets.get(scope="capstone_scope", key="openai_api_key")` inside the Databricks environment check.

### Step 3 ‚Äî Create a shared cluster (or use serverless)

Go to **Compute ‚Üí Create Compute**. For this pipeline, a single-node cluster works fine (no distributed Spark needed):

- Runtime: **14.x LTS ML** (includes many ML libraries pre-installed)
- Node type: `i3.xlarge` or `m5.xlarge` (cost-effective)
- Enable **"Terminate after X minutes of inactivity"**

For the reranker task specifically, you may want a GPU cluster (`g4dn.xlarge`) since `cross-encoder/ms-marco-MiniLM-L-6-v2` runs faster on GPU, but CPU works fine too.

### Step 4 ‚Äî Create Job A (Full Pipeline)

Go to **Workflows ‚Üí Create Job**. Name it `Full RAG Pipeline`.

Add tasks in this order:

**Task 1 ‚Äî download_pdfs**
- Type: `Python script`
- Source: Git repo
- Path: `notebooks/DataEngineeringDay1.py`
- Cluster: your shared cluster
- No dependencies

**Task 2 ‚Äî extract_clean_text**
- Type: `Python script`
- Path: `notebooks/DataEngineeringDay2.py`
- Depends on: `download_pdfs`

**Task 3 ‚Äî chunk_text**
- Type: `Python script`
- Path: `notebooks/DataEngineeringDay3.py`
- Depends on: `extract_clean_text`

**Task 4 ‚Äî embed_chunks**
- Type: `Python script`
- Path: `retrieval/01_embed_chunks.py`
- Depends on: `chunk_text`

**Task 5a ‚Äî build_faiss_index** and **Task 5b ‚Äî build_bm25_index**
- Both depend on `embed_chunks`, but they depend on each other ‚Äî Databricks will run them in parallel automatically when you set both to depend only on `embed_chunks`

**Task 6 ‚Äî hybrid_retriever**
- Depends on: `build_faiss_index` AND `build_bm25_index` (both must complete)

**Task 7 ‚Äî reranker**
- Depends on: `hybrid_retriever`

Step 7 ‚Äî Running and monitoring
For Job A, click Run Now once to build all your indexes. This is a one-time setup (or re-run whenever your source data changes).
For Job B, click Run Now or trigger via the REST API:
bashcurl -X POST https://<your-workspace>.azuredatabricks.net/api/2.1/jobs/runs/submit \
  -H "Authorization: Bearer <token>" \

You can monitor both jobs under Workflows ‚Üí Job Runs, where you'll see task-level success/failure, logs, and timing for each stage.

Summary of the recommendation
Use Databricks Workflows with two jobs. You never need to manually open and run files one by one. Job A is your build pipeline (run once or on a schedule when source data updates), and Job B is your query pipeline (run on demand or via API). The S3 bucket is the shared state layer between the two jobs ‚Äî Job A writes indexes to S3, Job B reads from them
